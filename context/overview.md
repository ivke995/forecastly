# Overview

Forecastly is a modern weather forecasting web application built with Next.js 16, TypeScript, Tailwind CSS, and the App Router.

The active route tree lives in the repository-root `app/` directory. The root layout provides the app-wide Forecastly shell: static Forecastly metadata, a text-only header, and a mobile-first content container capped at about 1200px. The home page (`app/page.tsx`) is a `'use client'` component that integrates `CitySearch` for city selection and calls `getWeatherForecast` on selection. It manages loading, error, success, and persisted temperature-unit display states. Before a city is selected, the temporary Forecastly onboarding placeholder is shown. After selection, the city `displayName`, an accessible Celsius/Fahrenheit toggle, the `CurrentWeatherCard` component (current temperature, feels-like, humidity, wind, condition, selected unit, and optional current-risk badges), the `WeatherRecommendations` component (deterministic advice derived from the normalized forecast), the `HourlyForecast` component (24 hourly entries plus temperature/precipitation trend chart using the selected unit), and the `DailyForecast` component (7-day responsive forecast plus high/low temperature and precipitation trend chart using the selected unit) are displayed. Unit switching is client-side display conversion and does not refetch the forecast.

Reusable Forecastly app-domain TypeScript interfaces live under the repository-root `types/` directory. The current domain model layer includes city-related interfaces in `types/city.ts` and weather forecast interfaces in `types/weather.ts`; weather conditions include numeric code, label, description, emoji, and severity. These are type-only contracts with no runtime data fetching or UI behavior.

Runtime external-data services and helpers live under the repository-root `lib/` directory and remain UI-free. `lib/geocoding.ts` exposes `searchCities(query: string)` for Open-Meteo city search, returning normalized `CitySearchResult` rows or `[]` for empty/no-result searches and throwing `GeocodingError` for provider/network/payload failures. `lib/weather-codes.ts` exposes `getWeatherCondition(code: number)` for centralized WMO/Open-Meteo weather-code mapping with safe unknown-code fallback. `lib/open-meteo.ts` exposes `getWeatherForecast(city: City)` for seven-day Open-Meteo forecasts, validating unknown provider payloads, using the centralized weather-code mapper, normalizing valid data into `WeatherForecast`, and throwing `WeatherForecastError` for API/network/parse/malformed-payload failures. `lib/weather-recommendations.ts` exposes `getWeatherRecommendations(forecast)` for UI-free, deterministic advice items shown by the home page recommendations card. `lib/weather-risk-badges.ts` exposes `getCurrentWeatherRiskBadges(current)` for UI-free, deterministic current-weather risk badge metadata rendered by `CurrentWeatherCard` when current risks exist.

The project uses [shadcn/ui](https://ui.shadcn.com/) (`base-nova` style, v4) as its UI component library. shadcn was initialized with the Next.js template and provides reusable, unstyled primitives under `components/ui/`. Currently available components: `Card` (and sub-components), `Button`. Additional components can be added via `npx shadcn@latest add <component>`. See [shadcn-ui.md](../context/components/shadcn-ui.md) for details.
